Основные основные комманды

git add . - добавить неотслеживаемые изменения.
git commit -m "add something" - фиксация изменений
git commit -a -m то можно не прописывать git add. НО ПРИ ДАННОЙ КОМАНДЕ НЕ ЗАПИШУТСЯ UNTRACKED файлы.
git push origin master - отправить на уделенный репозиторий
git status -узнать текущий статус репозитория
git log - узнать о всех коммитах
git remote -v - узнать какие origin (Для СОЗДАНИЯ git remote add origin https//yourlinck.ru) (Для удаления git remote remove origin)
git branch -r на какой ветке мы находимся (+ NAME_BRANCH - переход на указанную ветку) (+ - d удалить ветку в локальном репозитории) (+ - D удалитьветку в улаоенном репозитории)
git mkdir forExampleNameDir - создать папку, необходимо написать ее название
git init - Инициализировать git в папке, делается первый раз, создается скрытая папка .git
HEAD^^ или HEAD~2 перемещение указателя HEAD или хеш: alkn54fag2aefh2ns5jy2s5htrfgjd256dxgfhsn25 (хеш генерируется гитом самостоятельно)

Основные команды

git diff


git diff - Покажет ращницу между ТЕКУЩИМ НЕОТСЛЕЖИВАЕМЫМ состоянием репозитория и последним снимком репозитория.
git diff -- staged - Покажет ращницу между ТЕКЩИМ ОТСЛЕЖИВАЕМЫМ состоянием репозитория и последним снимком репозитория.
git diff sfgsb35sfgh15dfgs354av1f - (COMMIT_ID) Покажет ращницу между ТЕКУЩИМ состоянием репозитория и УКАЗАННЫМ снимком репозитория.

git reset


Команда используется для переписывания истории
git reset --soft возвращает проект к указанному коммиту, при этом переводит все коммиты после указанного в ОТСЛЕЖИВАЕМУЮ ЗОНУ
git reset --mixed возвращает проект к указанному коммиту, при этом переводит все коммиты после указанного в НЕОТСЛЕЖИВАЕМУЮ зону
git reset --hard врзвращает проект к указанному коммиту, при этом полностю улаояет все коммиты прсое указанного. УДАЛЯЕТ КОММИТЫ БЕЗВОЗВРАТНО!!!

git checkout


Используется для перемещения между коммитами, версиями отдельных фалов, ветками, если мы хотим посмотреть как выглядел проект в каком-то снимке в прошлом. Коммиты не удалятся. Все мзенения не сохраняются (Удаляются сборщиком мусора).
git checkout master - возврат на последнюю версию.
git checkout adslnb49uhva9etrovn34us03ffafn944b9494 -- file1 file2
git checkout название ветки, позволит переместиться на другую ветку.
+

git commit --ammend -m


Дополняет последний коммит, добавляя в него свежие изменения. Таже мнеяет сообщение последнего коммита. Новй коммит не создается, перепичывается "новый коммит".

git clean


Удаляет UNTRAKT файлы. По этому для начала нужно прописать команду:
git clean -n Данная команда покажет, что при git clean удалится
git clean -f Уалит изменения связанные с untract файлов.
